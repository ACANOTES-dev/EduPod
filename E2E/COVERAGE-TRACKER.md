### Report Cards module overhaul (2026-04)

The Report Cards module was completely overhauled between the initial coverage estimate (5 pages) and the admin/teacher specs written on 2026-04-10. The revamped module now spans **14 unique routes** split across `/report-cards/*` and `/report-comments/*`:

1. `/report-cards` — Dashboard with period selector, quick action tiles (4 for admin, 2 for teacher), live run panel, analytics snapshot, classes-by-year-group grid
2. `/report-cards/{classId}` — Per-class grade matrix with period filter + grade/score toggle + top-rank badges
3. `/report-cards/settings` — Tenant-wide settings: display defaults, comment gate, personal info defaults, default template, principal details + signature upload
4. `/report-cards/generate` — 6-step generation wizard (scope / period / template / fields / comment gate / review) + polling + terminal states
5. `/report-cards/library` — Grouped library with by-run / by-year-group / by-class views, row-level publish/unpublish/delete, bundle downloads
6. `/report-cards/analytics` — Summary cards, class comparison chart, per-class generation progress
7. `/report-cards/requests` — Teacher requests list with Pending/All admin tabs
8. `/report-cards/requests/{id}` — Request detail with Approve & Open / Auto-approve / Reject flows
9. `/report-cards/requests/new` — Teacher-only new request form
10. `/report-comments` — Landing page with window banner + homeroom cards + subject cards derived from competencies × curriculum matrix
11. `/report-comments/overall/{classId}` — Overall comments editor with autosave + finalise/unfinalise

### Finance module fix sweep (2026-04-12)

The initial admin walkthrough of the Finance module flagged **48 distinct issues** in §97 of the admin spec: 22 translation-debt items, 22 functional inconsistencies, and 4 documentation/testability gaps. Within the same day a follow-up fix sweep resolved every one of them — commit `24073202` (35 files, +3,141 / −388 lines) deployed directly to `nhqs.edupod.app` and Playwright-verified.

**Highlights of what changed:**

- 67 new keys added to `finance` namespace in `apps/web/messages/{en,ar}.json` (tab labels, method labels, action modal copy, toast messages, loading button labels, aging bucket labels, entry-type badge labels, ICU-interpolated audit-trail descriptions, refund reject modal copy, and more)
- Invoice list gains a status filter + 300ms debounced search + URL `?status=` hydration
- Debt-breakdown writes the bucket filter back to the URL via `router.replace`
- Dashboard OverdueInvoices + debt-breakdown rows now navigate via `router.push` instead of `window.location.assign`
- Dashboard KPI cards, recent payments, top debtors, aging totals all render through `<CurrencyDisplay currency_code={useTenantCurrency()}>` — tenant symbol (`€`, `$`, etc.) visible everywhere instead of bare numbers
- HouseholdSelector fetches the selected household by id when it's not in the currently-loaded page
- Refund Reject now opens a dedicated modal with a required comment textarea — previously 100% of reject attempts 400'd because the client sent an empty body
- Refund approve/reject/execute all emit `toast.success` / `toast.error` with translated messages
- `RefundsService.execute` wires `StripeService.processRefund` for stripe-method payments with compensating `failed` state on Stripe errors
- Payment-plans empty state only renders on the `active` tab
- Late-fee-configs + recurring-configs now use `PATCH` for updates (were `PUT`)
- `updateDiscountSchema` carries the create-time percent-cap + auto-apply-condition refinements
- `feeGenerationPreview/ConfirmSchema` enforce date ordering via `.refine()`
- `reportQuerySchema` moved from the controller to `@school/shared`
- `GET /payments/staff` tightened from `finance.view` → `finance.manage`
- Receipt PDF filename uses `receipt_number` with UUID fallback
- DiscountForm refinement messages sourced from a `makeDiscountFormSchema({ percentMax, needCondition })` factory so Arabic users see Arabic validation messages

Four scope items were explicitly deferred and documented in §97 of the spec ("Remaining known issues"): bulk export as JSON (not streamed), notification delivery wiring for payment reminders, cross-module shared state-machine files, and the four pre-existing unrelated jest failures in `finance-dashboard`/`payments`/`stripe` specs that fail on `main` independently of this fix sweep.

Verification:

- `turbo type-check` — clean on `@school/shared`, `@school/web`, `@school/api`
- `eslint` on finance sources — 0 errors, 50 warnings (all pre-existing cross-module-import + max-lines + exhaustive-deps)
- `jest src/modules/finance` — 35/39 suites pass (4 pre-existing failures unchanged)
- Playwright on prod — dashboard `€83,000.01 / €34,601.01 / €48,399.00` on KPI tiles, invoice list `?status=issued` hydrates filter + returns 11 rows, invoice detail shows translated metric labels + status badge, debt-breakdown `50%+` click writes `?bucket=50_plus` to URL, statements ledger shows translated `Invoice` entry-type badge, fee-structures frequency reads `One-off`, refund create modal shows `Method: Cash` (translated), zero console errors
