# SW-3A: NEPS Referrals

## What This Sub-Phase Delivers

1. **Referral service** — full CRUD and lifecycle management for external referrals (NEPS, CAMHS, Tusla family support, Jigsaw, Pieta House, other)
2. **Referral pre-population service** — cross-module data aggregation that snapshots attendance, gradebook, behaviour, intervention, and parent-contact data at referral creation time
3. **Referral recommendation tracking** — when the NEPS report returns, recommendations become trackable action items with their own lifecycle
4. **NEPS visit calendar** — lightweight tracking of psychologist visits, students seen, outcomes
5. **Waitlist visibility** — elapsed time since referral submission, surfaced in SST meetings and principal reports

## State Machines

### Referral Lifecycle

```
draft -> submitted -> acknowledged -> assessment_scheduled -> assessment_complete -> report_received -> recommendations_implemented
         \                                                                          /
          \-------------------------> withdrawn (from any active state) <-----------/
```

| From | To | Trigger | Side Effects |
|---|---|---|---|
| `draft` | `submitted` | User submits referral | `referral_submitted` audit event; `submitted_at` stamped; waitlist clock starts |
| `submitted` | `acknowledged` | External agency acknowledges receipt | `referral_acknowledged` audit event; `acknowledged_at` stamped |
| `acknowledged` | `assessment_scheduled` | Assessment date set | `referral_assessment_scheduled` audit event; `assessment_scheduled_date` set |
| `assessment_scheduled` | `assessment_complete` | Assessment done | `referral_assessment_complete` audit event; `assessment_completed_at` stamped |
| `assessment_complete` | `report_received` | Report entered | `referral_report_received` audit event; `report_received_at` stamped |
| `report_received` | `recommendations_implemented` | All recommendations implemented or marked n/a | `referral_recommendations_implemented` audit event |
| any active | `withdrawn` | User withdraws referral | `referral_withdrawn` audit event with reason |

### Recommendation Lifecycle

```
pending -> in_progress -> implemented
                       -> not_applicable
```

| From | To | Trigger | Side Effects |
|---|---|---|---|
| `pending` | `in_progress` | Staff begins working on it | `recommendation_status_changed` audit event |
| `in_progress` | `implemented` | Staff marks complete | `recommendation_status_changed` audit event |
| `in_progress` | `not_applicable` | Staff determines not relevant | `recommendation_status_changed` audit event; `status_notes` required |
| `pending` | `not_applicable` | Staff determines not relevant | `recommendation_status_changed` audit event; `status_notes` required |

## Cross-Module Integration Points

| Imported From | Service/Method | Purpose | Access Mode |
|---|---|---|---|
| `AttendanceModule` | `AttendanceService.getStudentSummary()` or raw Prisma query | Attendance percentage, chronic absence flag | Read-only |
| `GradebookModule` | `GradesService` or raw Prisma query | Current grades per subject, trend calculation | Read-only |
| `BehaviourModule` | `BehaviourService` or raw Prisma query | Incident counts by category | Read-only |
| Pastoral (own) | `InterventionService` (from SW-2B) | Prior interventions with outcomes and continuum levels | Read-only |
| Pastoral (own) | `ConcernService` (from SW-1B) | Parent contact records from concern follow-ups | Read-only |
| Pastoral (own) | `PastoralAuditService` (from SW-1A) | Write audit events for every status transition | Write |

**Module wiring:** The `PastoralModule` must import `AttendanceModule`, `GradebookModule`, and `BehaviourModule` (or use `forwardRef` if circular). The pre-populate service reads from these modules but never writes to them.

**Important:** The pre-populated data is a snapshot. It is generated once at the user's request and stored as JSONB. It does not live-update. If the user wants a fresh snapshot, they call the `pre-populate` endpoint again (which overwrites the previous snapshot on a draft referral only).
