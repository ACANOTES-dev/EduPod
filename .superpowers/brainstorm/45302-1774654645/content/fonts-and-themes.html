Each shows light mode (top) and dark mode (bottom). All use CSS custom properties — no hardcoded colours, ever.

About your white-text-on-light-mode bug: All three options use design tokens (CSS variables) that swap per theme. Buttons use var(--btn-text) not #fff. This structurally prevents the invisible-button problem.
