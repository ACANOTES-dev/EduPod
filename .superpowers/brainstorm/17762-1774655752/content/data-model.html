Section 1: Data Model

New tables for the Early Warning module — all tenant-scoped with RLS

Entity Relationship Overview

Early Warning Data Model
student_risk_profiles
One row per student per academic year. The "current state" table.
id UUID PK tenant_id UUID FK NOT NULL student_id UUID FK NOT NULL academic_year_id UUID FK NOT NULL composite_score NUMERIC(5,2) 0-100 risk_tier ENUM green/yellow/amber/red tier_entered_at TIMESTAMPTZ attendance_score NUMERIC(5,2) grades_score NUMERIC(5,2) behaviour_score NUMERIC(5,2) wellbeing_score NUMERIC(5,2) engagement_score NUMERIC(5,2) signal_summary_json JSONB — NL text + data points trend_json JSONB — last 30 daily scores assigned_to_user_id UUID FK NULLABLE assigned_at TIMESTAMPTZ NULLABLE last_computed_at TIMESTAMPTZ NOT NULL created_at TIMESTAMPTZ DEFAULT now() updated_at TIMESTAMPTZ @updatedAt
UNIQUE (tenant_id, student_id, academic_year_id)  |  INDEX (tenant_id, risk_tier)  |  INDEX (tenant_id, composite_score DESC)
student_risk_signals
Append-only audit trail. One row per signal detection event. Enables trend analysis and historical validation.
id UUID PK tenant_id UUID FK NOT NULL student_id UUID FK NOT NULL academic_year_id UUID FK NOT NULL domain ENUM attendance/grades/behaviour/wellbeing/engagement signal_type VARCHAR — e.g. 'consecutive_absences' severity ENUM low/medium/high/critical score_contribution NUMERIC(5,2) — points added details_json JSONB — signal-specific data source_entity_type VARCHAR — e.g. 'AttendancePatternAlert' source_entity_id UUID — FK to source record detected_at TIMESTAMPTZ NOT NULL created_at TIMESTAMPTZ DEFAULT now()
NO updated_at (append-only)  |  INDEX (tenant_id, student_id, detected_at DESC)  |  INDEX (tenant_id, domain, detected_at DESC)
early_warning_tier_transitions
Append-only. Logs every tier change. Drives notifications, hysteresis checks, and historical analysis.
id UUID PK tenant_id UUID FK NOT NULL student_id UUID FK NOT NULL profile_id UUID FK NOT NULL from_tier ENUM green/yellow/amber/red NULLABLE to_tier ENUM green/yellow/amber/red NOT NULL composite_score NUMERIC(5,2) — score at transition trigger_signals_json JSONB — which signals caused the change routed_to_user_id UUID FK NULLABLE notification_id UUID FK NULLABLE — links to Notification transitioned_at TIMESTAMPTZ NOT NULL created_at TIMESTAMPTZ DEFAULT now()
NO updated_at (append-only)  |  INDEX (tenant_id, student_id, transitioned_at DESC)  |  INDEX (tenant_id, to_tier, transitioned_at DESC)
early_warning_configs
Per-tenant configuration. Weights, thresholds, routing rules, hysteresis buffer, digest schedule.
id UUID PK tenant_id UUID FK NOT NULL UNIQUE is_enabled BOOLEAN DEFAULT false weights_json JSONB — {attendance: 25, grades: 25, ...} thresholds_json JSONB — {green: 0, yellow: 30, amber: 50, red: 75} hysteresis_buffer INTEGER DEFAULT 10 — points below threshold to downgrade routing_rules_json JSONB — tier → role/user mapping digest_day INTEGER DEFAULT 1 — day of week (1=Mon) digest_recipients_json JSONB — user IDs or role filters high_severity_events_json JSONB — which events trigger intraday recompute created_at TIMESTAMPTZ DEFAULT now() updated_at TIMESTAMPTZ @updatedAt
UNIQUE (tenant_id)  |  All JSONB fields have Zod schemas with .default() values
Colour Legend
White — identifiers & timestamps Green — classification / status fields Cyan — score / metric fields Purple — JSONB structured data Pink — routing / assignment fields Gold border — mutable (updated daily) Cyan border — append-only audit trail Pink border — append-only transitions Green border — configuration (rarely changes)

Key Design Decisions

student_risk_profiles — one row per student per academic year. Updated in-place during each computation cycle. Stores the current state including composite score, 5 sub-scores, NL summary, and 30-day trend array. This is what the dashboard queries.
student_risk_signals — append-only log of every individual signal detected (e.g., "3 consecutive absences", "grade dropped below class mean by 15%"). Enables drill-down from a score to the specific data points that produced it. Source entity FK links back to the original record in each domain.
early_warning_tier_transitions — append-only log of every tier change. Drives hysteresis (check last transition before downgrading), notification dispatch (only notify on actual tier changes), and historical analysis (how long do students spend in each tier?).
early_warning_configs — one row per tenant. All JSONB fields have Zod schemas with .default() so new tenants get sensible defaults without migration. Follows the existing TenantSetting pattern but as a dedicated table for type safety.
No separate cohort table — cohort/heatmap views are computed on-the-fly by aggregating student_risk_profiles grouped by year_group, class, or subject. Avoids maintaining another denormalized table. If performance becomes an issue, we add a materialized view later.
