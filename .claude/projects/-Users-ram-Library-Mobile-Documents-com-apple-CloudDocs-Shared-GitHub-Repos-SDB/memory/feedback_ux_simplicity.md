---
name: UX Simplicity — Hide the Complexity
description: High technicality hidden from users. Simple, friendly interfaces. Complexity = dropout.
type: feedback
---

The product is technically sophisticated but the interface must be simple and friendly. Users are teachers, parents, and school admins — not engineers.

**Why:** Complexity causes dropout. If a school admin needs training to use a feature, the feature has failed. The bar is "a teacher can figure this out in 30 seconds."

**How to apply:** Never expose internal complexity (IDs, technical status names, system terminology) in the UI. Use plain language. Prefer smart defaults over configuration screens. Progressive disclosure — show the simple version first, let power users dig deeper. Every screen should feel obvious.
