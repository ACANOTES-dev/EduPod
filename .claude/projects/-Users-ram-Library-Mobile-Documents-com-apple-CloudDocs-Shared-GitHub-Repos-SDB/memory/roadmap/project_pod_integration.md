**Priority:** Non-negotiable for primary market entry. Can be deferred if post-primary is the initial target.

**What it is:**
- POD (Primary Online Database) is the nationwide individualised database of all primary school pupils
- Maintained by the Department of Education
- Used for: National School Annual Census (since September 2016), teacher allocation, capitation grants, pupil register
- Schools submit returns annually in October
- Tracks pupils through primary system and into post-primary

**What EduPod must do:**
- Student data sync to POD
- Annual October census submission
- Validation of student records before submission
- Handling of pupil transfers between schools via POD
- Roll number management

**Competitor status:**
- Aladdin: deep POD integration — this is their core competency serving 90% of primary schools
- Compass: supports POD (they target both primary and post-primary)
- VSware: primarily post-primary, P-POD focused

**Effort estimate:** Medium — similar to P-POD but different format and requirements specific to primary schools.

**How to apply:** Defer this if targeting post-primary first. Build when ready to enter the primary market. The primary market is heavily defended by Aladdin (90% share) so this is a lower priority than P-POD.
