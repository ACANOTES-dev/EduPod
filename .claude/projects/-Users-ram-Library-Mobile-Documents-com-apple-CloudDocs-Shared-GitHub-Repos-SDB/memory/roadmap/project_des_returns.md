**Priority:** Non-negotiable. Statutory annual requirement for all Irish schools.

**What it is:**
- **September Returns:** Schools generate encrypted data files (A, B, C, D, E) for DES submission annually. Includes staff data, class data, subject data, student enrolment data.
- **Form TL:** Teacher hour tracking — how many hours each teacher teaches per week by subject.
- **DTR Returns:** Subject code management and synchronisation with DES subject code lists.
- **Census Returns:** Validation of enrolment data for DES census.

**What EduPod must do:**
- Generate encrypted September Return files (A through E) in DES-specified format
- Generate Form TL from timetabling data (teacher hours per subject)
- Maintain DES subject code mappings
- DTR subject code sync
- Census return data validation
- Pre-submission validation to catch errors before upload

**Competitor status:**
- Compass: DES-compliant statutory returns with validation
- VSware: generates encrypted files A-E, Form TL, DTR — battle-tested across hundreds of schools for 10+ years

**Effort estimate:** Medium — requires understanding DES file formats and encryption requirements. EduPod already has staff, class, subject, and timetabling data. Main work is format compliance and encryption.

**How to apply:** Must be built before approaching any Irish school. The Form TL generation particularly benefits from EduPod's detailed timetabling data — teacher hours by subject are already tracked.
