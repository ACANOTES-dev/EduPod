name: Pre-Launch Item Routing
description: Any "remember before go-live" instruction goes to Manuals/PRE-LAUNCH-CHECKLIST.md Part 5, not memory
type: feedback
---

When the user says "remember this before we go live", "before launch", "pre-launch item", or anything about needing to do something before tenants are onboarded — it goes to `Manuals/PRE-LAUNCH-CHECKLIST.md` → Part 5 (Deferred Items).

**Why:** Pre-launch items need to be tracked in a single, reviewable checklist — not scattered across memory files that may get lost or go stale (which has already happened).

**How to apply:** Never store pre-launch items as memory files. Always add them to the pre-launch checklist with: what it is, where it was flagged, today's date, and relevant notes. The rule at `.claude/rules/pre-launch-tracking.md` governs this.
