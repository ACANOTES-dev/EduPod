- Unit tests with coverage enforcement
- Test coverage gate
- Database setup + migrations + seed
- Integration/e2e tests
- Build

**deploy job**: SSH to production, runs `scripts/deploy-production.sh`. Concurrency group prevents parallel deploys.

**integration job**: Separate Postgres (5553) + Redis (5554) with `school_platform` DB. Includes restore drill verification.

**visual job**: Playwright visual smoke tests with artifact upload on failure.

### Production Setup

- PM2 with 3 processes: api (cluster, 750M), web (cluster, 1G), worker (fork, 750M)
- PgBouncer in transaction mode (pool size 20, max 200 clients)
- Redis 7, Meilisearch for search
- Docker Compose for local dev (postgres, pgbouncer, redis, meilisearch)

### No deploy workflow separate from CI — deploy is embedded in the CI workflow, triggered on push to main after CI passes.
